一、 项目概述 (Executive Summary)
1.1 产品定位
本产品是一款专注“婚纱摄影”垂直领域的专业级AI生图Web平台。产品旨在彻底颠覆传统婚纱摄影“高客单价、高体力消耗、高沟通成本”的痛点，通过将复杂的AI底层技术（特征提取、空间控制、提示词工程、图像重绘）封装为极简的“傻瓜式”用户体验，为准新人提供影楼级、4K/8K画质、100%忠于本人特征且极具情绪张力的数字婚纱照。

1.2 核心价值主张 (Value Proposition)
“把物理世界的风吹日晒与假笑，交给AI管线；把最完美的数字记忆与视觉惊艳，留给新人。”
- 对用户：0提示词门槛，像点外卖一样选“视觉套餐”，一杯咖啡的时间获取完美大片。
- 对系统：铁打的管线，流水的模型。内聚所有专业摄影知识，底层模型解耦，实现真正的高良品率（0废片交付）。

二、 目标用户与痛点分析
2.1 目标用户画像
- 备婚期情侣（主力）：预算有限、时间紧迫，或对传统影楼流水线风格感到厌倦的95后/00后。
- “社恐”及“懒人”群体：极度排斥镜头前尴尬摆拍、不愿经历严苛身材管理和风吹日晒的用户。
- 追求极致视觉体验者：希望实现现实中无法完成的极限旅拍（如极光、深海、赛博朋克、宇宙星空）的情侣。

2.2 核心痛点与本产品解法
- 痛点1：【美化悖论】写实太丑，美颜太假，缺乏传统影楼的“化妆缓冲期”。
  解法：引入“AI试妆间”，通过轻/中/重度三档妆造盲测，让用户自行选择“理想自我”的权重阈值。
- 痛点2：【情绪调度】AI生成的双人照往往表情僵硬，缺乏婚纱照灵魂（眼神拉丝）。
  解法：双模型协作。用Nano Banana Pro主打画质与ID，用gpt-image-1.5重绘面部情绪，注入微表情。
- 痛点3：【物理坍塌】双人紧密互动（拥抱、亲吻）极易导致手指畸形、肢体穿模。
  解法：构建“VLM视觉反思与自我修复循环”，后台自动找茬、自动修复，阻断废片流向用户。

三、 核心流水线架构设计 (The Virtual Studio Pipeline)
本产品采用“Agentic Workflow（智能体工作流）”，将流程拆分为五个解耦模块：

3.1 模块一：数字人质押（特征提取层）
- 动作：用户上传5-10张高清日常照。
- 系统处理：自动提取Face-ID Embeddings与骨相/皮相向量数据。
- 交互环节：弹出“AI试妆间”，生成3张单人测试图供用户二选一，系统据此锁定LoRA/Style美化权重。

3.2 模块二：中枢调度与参数编译（摄影导演层）
- 动作：用户在前端选择“视觉套餐”（如：王家卫胶片风）。
- 系统处理：架构调度器（LLM）将套餐转化为“摄影结构体（Camera Schema）”。
  * 镜头语言：如 85mm, f/1.2。
  * 光影逻辑：如 霓虹侧逆光, 低反差。
  * 编译下发：将这些参数自动翻译为底层模型懂的Prompt指令。

3.3 模块三：空间控制与基础生成（物理防撞层）
- 系统处理：加载预设的双人互动ControlNet（骨骼图）与Depth（深度图），明确前后景遮挡关系。
- 渲染：调用主摄影师模型（Gemini Nano Banana Pro）结合数字人特征，进行原生4K底图渲染。

3.4 模块四：AI视觉反思与自我修复（质检暗房层 —— 核心技术壁垒）
- 动作1（找茬）：将底图输入VLM（视觉大模型），执行严苛质检指令：“检查手指数量、肢体穿模、表情僵硬度，输出错误坐标JSON”。
- 动作2（修复）：
  * 若手指/物理结构报错 -> 生成Mask，调用 Nano Banana Pro 的局部重绘（Inpainting）修补物理错误。
  * 若表情/情绪报错 -> 生成面部Mask，调用 gpt-image-1.5 进行I2I重绘，注入“深情、喜悦”等情感指令。
- 循环：内部最多循环3次，直至VLM判定合格。

3.5 模块五：魔法后期与工业输出（交付层）
- 前端呈现：向用户展示完美成片。
- 用户微调：提供极简的“魔法笔刷”（如：圈选裙摆点击“换成鱼尾裙”，圈选背景路人点击“消除”）。
- 最终输出：一键执行8K超分放大（Upscaling），提供可用于冲印实体海报的无损大文件。

四、 Web端用户操作路径 (User Journey)
极致的“去AI化”体验，全流程隐藏技术参数：[Step 1: 建立档案] 
页面提示：“欢迎来到您的私人数字影棚”。引导用户分别上传男女双方各5张日常清晰照片。

[Step 2: AI试妆] 
页面展示男女各3种状态的头像（素颜质感、精致新娘/新郎妆、完美骨相微调）。
提示：“哪一个是您今天想要的状态？” -> 点击确认。[Step 3: 挑选套餐（核心变现路径）] 
像刷小红书或Netflix一样，展示精美的套餐封面（无文本输入框）。
分类示例：《极简高定棚拍》、《冰岛黑沙滩史诗》、《中式赛博朋克》、《法式街角胶片》。
选择后点击 -> “开始拍摄”。[Step 4: 沉浸式等待] 
（后台正在疯狂执行模块三和模块四的生成与自我修复循环）。
前端展示文案安抚：“正在为您布置法式街角的灯光...”、“正在调整新娘的头纱质感...”、“摄影师正在抓拍最完美的笑容...”。（耗时约30-60秒）[Step 5: 审片与交付] 
大图震撼呈现。
提供三个简单按钮：【完美，去冲印（下载8K图）】、【换套衣服（圈选重绘）】、【再拍一张（重新生成）】。

五、 商业模式与产品规划
5.1 盈利模式 (Monetization)
- 体验引流：首次注册可免费生成1张带水印的标清图（培养Aha Moment）。
- 基础套餐买断制（单次付费）：99元/199元，可解锁特定主题（如极简系列），包含20张4K精修成片+2张8K海报级输出。
- SVIP无限畅拍订阅制：599元/季度。适合重度“发朋友圈”需求或试错需求高的用户，解锁全站所有极端/高难度场景套餐（如星空、水下、特效类），支持无限次局部重绘修改。
- 实体冲印增值服务：与线下高品质冲印厂合作，直接在Web端一键下单相册、挂画、迎宾海报，客单价提升至500-1000元。

5.2 演进路线图 (Roadmap)
- Phase 1 (MVP内测期)：跑通 Nano Banana Pro 单模型链路，手工介入质检，完成创始团队（即提出者）的婚纱照0号样本打样。
- Phase 2 (1.0 上线期)：实装双核调度（接入gpt-image-1.5）与VLM自我修复管线，上线Web端，推出50个标准化视觉套餐，启动小红书/抖音等社交媒体的素材裂变营销。
- Phase 3 (2.0 破圈期)：开放“用户自定义场景”高级模式（将大语言模型转换为底层Prompt）；研发“AI婚礼微电影”功能（基于图生视频技术，让婚纱照动起来）。

六、 风险评估与对策
- 技术风险：底层API接口成本波动及生图时间过长。
  * 对策：通过VLM前置预判成功率，减少无效重绘；建立自有的高频并发队列调度系统。
- 隐私风险：用户上传的面部数据敏感。
  * 对策：承诺生成结束后24小时内自动销毁源照片及Face-ID向量特征，页面出具显式隐私协议，建立数据安全壁垒。
- 审美疲劳风险：固定套餐容易让用户看腻。
  * 对策：产品运营需保持高频上新，紧盯社交媒体流行趋势（如近期流行的特定电影风），每周推出限定版“主题盲盒”。
