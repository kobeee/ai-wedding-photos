# VLM 质检管线

> AI 视觉反思与自我修复系统，对应 [[PRD分析#模块四：AI视觉反思与自我修复]] ⭐核心壁垒。

---

## 概述

生成的底图不直接交付用户，而是经过 VLM（Vision Language Model）视觉反思循环，自动检测并修复常见生成缺陷。

**核心理念**：AI 自己给自己"找茬"，然后自己修。

---

## 检查维度

| 维度 | 检查内容 | 错误类型 | 修复模型 |
|------|----------|----------|----------|
| 手指 | 手指数量、形态、关节 | 物理错误 | [[Nano-Banana-Pro]] |
| 穿模 | 衣物穿透身体、饰品错位 | 物理错误 | [[Nano-Banana-Pro]] |
| 五官 | 眼睛对称、嘴巴形态 | 物理错误 | [[Nano-Banana-Pro]] |
| 表情 | 微笑自然度、情绪一致性 | 情绪错误 | [[Nano-Banana-Pro]]（当前运行态） |
| 肢体 | 身体比例、姿态合理性 | 物理错误 | [[Nano-Banana-Pro]] |
| 光影 | 光源一致性、阴影方向 | 物理错误 | [[Nano-Banana-Pro]] |

---

## 反思循环流程

```
        ┌─────────────────────────────────────┐
        │                                     │
        ▼                                     │
   [底图生成]                                  │
   (Nano Banana Pro)                          │
        │                                     │
        ▼                                     │
   [VLM 视觉检查]                              │
   "检查这张婚纱照是否存在以下问题：              │
    手指异常/穿模/表情不自然/..."                │
        │                                     │
    ┌───┴───┐                                 │
    │       │                                 │
  合格    不合格                               │
    │       │                                 │
    ▼       ▼                                 │
  交付   [分类错误]                             │
          │                                   │
     ┌────┴────┐                              │
     │         │                              │
  物理错误  情绪错误                            │
     │         │                              │
     ▼         ▼                              │
  Nano修复   Nano修复                           │
  Prompt     Prompt                            │
     │         │                              │
     └────┬────┘                              │
          │                                   │
          ▼                                   │
     [轮次计数]                                │
     round < 3 ? ──是──────────────────────────┘
          │
          否
          │
          ▼
     [强制交付]
     (标记质量分数)
```

## VLM Prompt 设计

### 质检 Prompt

```
你是一位专业的婚纱摄影质检师。请仔细检查这张AI生成的婚纱照，
按以下维度评估并报告问题：

1. 手指：数量是否正确（每只手5个）？形态是否自然？
2. 穿模：衣物/饰品是否穿透身体？
3. 表情：新人的表情是否自然、幸福？
4. 五官：眼睛、嘴巴、鼻子是否对称？
5. 肢体：身体比例是否协调？关节是否合理？
6. 光影：光源方向是否一致？阴影是否合理？

输出格式：
{
  "pass": true/false,
  "issues": [
    {
      "type": "physical" | "emotional",
      "category": "手指/穿模/表情/五官/肢体/光影",
      "severity": "high/medium/low",
      "description": "问题描述",
      "region": "问题区域坐标或描述"
    }
  ],
  "quality_score": 0.0-1.0
}
```

### 修复 Prompt 生成

根据 VLM 检出的问题，自动生成修复用的 Prompt：

- **物理错误** → 生成 [[Nano-Banana-Pro#图生图]] repair prompt，带当前成片与身份参考修复
- **情绪错误** → 当前同样走 [[Nano-Banana-Pro#图生图]] repair prompt，聚焦表情、眼神、情绪氛围

> 说明：GPT 遮罩重绘仍保留为后续实验位，但当前实现没有稳定 mask 来源，因此不在默认链路中启用。

---

## 循环控制

| 参数 | 值 | 说明 |
|------|------|------|
| 最大循环次数 | 3 | 超过则强制交付 |
| 合格阈值 | quality_score >= 0.85 | 分数达标则通过 |
| 高严重度容忍 | 0 | high 级别问题必须修复 |
| 修复超时 | 30s/轮 | 单轮修复超时则跳过 |

---

## 质量分数

| 分数范围 | 等级 | 处理 |
|----------|------|------|
| 0.95-1.0 | 优秀 | 直接交付 |
| 0.85-0.94 | 合格 | 交付，标记为"良好" |
| 0.70-0.84 | 需改善 | 进入修复循环 |
| < 0.70 | 不合格 | 触发重新生成 |

---

## 与等待页面的关系

VLM 质检循环的进度通过 [[API设计#GET /api/generate/{task_id}/status]] 返回给前端。

前端 [[页面清单#Waiting Page]] 展示的进度步骤：

```
生成底图 → 质量检查 → 优化修复 → 完成
```

修复轮次 > 1 时，文案切换为"正在精雕细琢..."以安抚用户。
