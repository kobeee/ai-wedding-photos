# 身材比例锚定方案

## 问题

AI生图时，双人婚纱照的身高差和身材比例完全靠模型猜测。真人拍照中摄影师眼前就是真人，比例天然准确；但AI只看到参考图的像素，不知道真实身高，两人合照时身高差、体型对比全靠瞎猜。

当前状态：
- 上传只校验格式和大小，无身材信息采集
- 前端引导只说"正面、侧面、全身"，无具体要求
- 生图 prompt 里只有一句"preserve body shape"，基本无效
- VLM 质检提到 `body proportions` 但只是被动评估，不主动引导

## 核心思路

**用AI的方式思考**：AI生图是"看图说话"，不是"看数字说话"。prompt里写"男175cm女163cm"模型不懂，但给一张两人站在一起的全身照，模型一眼就知道比例关系。

**方案**：要求用户上传至少1张双人全身日常合照，生图时作为第一张参考图传给 Gemini（支持最多14张参考图），配合 prompt 锚定指令，让模型以此为比例基准。

## 为什么不选其他方案

| 方案 | 否决原因 |
|------|----------|
| 让用户填身高体重 | 违反"零决策压力"原则，数字写进prompt模型也不一定理解 |
| VLM自动从单人照推断身高 | 没有参照物，单人照无法判断绝对身高 |
| AI推断后让用户确认/微调 | 多一个步骤，增加决策压力，且推断不一定准 |
| 人体骨骼检测/身高像素计算 | 过度工程，Gemini 2026版从参考图直接理解比例已经够了 |

## 实现要点

### 前端 Upload 页面
- 三段式布局：双人合照区（上方全宽）+ 新郎/新娘个人照（下方两列）
- 合照区限制最多1张，要求全身照
- 个人照片引导优化：必须包含全身照+面部特写
- 缺合照时"下一步"按钮 disabled

### 后端参考图筛选（核心改动）
- `reference_selector.py` 选择顺序改为：
  1. couple 合照 → 排第一位（比例锚定）
  2. bride 最佳单人照
  3. groom 最佳单人照
  4. 剩余名额按质量分补
- `MAX_TOTAL` 从 3 调到 4（合照占1个额外位置）

### Prompt 锚定指令
当有合照参考图时，`prompt_assembler.py` 追加：
> "The first reference image is a full-body couple photo establishing their real height difference and body proportions. Use it as the definitive scale anchor — preserve the exact height ratio and overall build between the two people."

### 不做的事
- 不加 body_reference 字段到 CreativeBrief
- 不改 nano_banana.py（参考图顺序由 selector 控制）
- 不改 GenerateRequest schema
- 不做合照自动人数检测（靠前端引导+role标记）

## 数据流

```
前端 Upload
  ├── coupleFiles[0] → POST /api/upload (role="couple")
  ├── maleFiles      → POST /api/upload (role="groom")
  └── femaleFiles    → POST /api/upload (role="bride")
        ↓
reference_selector.py
  └── primary[0] = couple合照 → bride最佳 → groom最佳 → 辅助
        ↓
prompt_assembler.py
  └── 追加 "scale anchor" 锚定指令
        ↓
nano_banana → Gemini API（合照作为第一张参考图）
```

## 风险

1. **Gemini是否真的尊重第一张参考图的比例？** — 需联调验证。备选：用VLM从合照提取粗略身高比数值写进prompt
2. **合照质量差**（模糊、裁切不全）可能比没有合照更糟 — 可对couple role做更严格的全身检测
3. **14张参考图上限**：合照占1个位置，总共才用4张，远低上限
