# 部署方案

> 完整的 Docker Compose + VPS + Nginx 部署链路。

---

## 部署拓扑

```
                    ┌─────────────────────────┐
                    │  VPS: 65.75.220.11      │
                    │                         │
用户浏览器 ──HTTPS──→│  Nginx (宿主机)          │
                    │    ↓ 反代                │
                    │  Docker Compose          │
                    │  ┌───────────────────┐   │
                    │  │ frontend:3080     │←──│── 静态资源 + SPA
                    │  │  (Nginx:80)       │   │
                    │  ├───────────────────┤   │
                    │  │ backend:8000      │←──│── /api/* 反代
                    │  │  (Uvicorn)        │   │
                    │  ├───────────────────┤   │
                    │  │ db (不暴露端口)    │   │
                    │  └───────────────────┘   │
                    └─────────────────────────┘
```

## 关键配置

### 域名

- **域名**: `wedding.escapemobius.cc`
- **SSL**: Let's Encrypt 自动续期
- **DNS**: A 记录指向 `65.75.220.11`

### VPS 信息

| 项目 | 值 |
|------|------|
| IP | 65.75.220.11 |
| 登录方式 | `ssh root@65.75.220.11` |
| 操作系统 | Ubuntu（推测） |
| 已有服务 | 多个，通过 Nginx 多站点路由 |

### Docker Compose 编排

> 配置文件详见 [[Docker配置]]

```yaml
services:
  frontend:
    build: ./docker/Dockerfile.frontend
    ports: ["3080:80"]
  backend:
    build: ./docker/Dockerfile.backend
    expose: ["8000"]  # 仅内部暴露
  db:
    # 端口不暴露
```

### Nginx 两层代理

1. **容器内 Nginx**（前端容器）
   - 服务前端静态资源
   - SPA fallback → `index.html`
   - 配置见 [[Nginx配置]]

2. **宿主机 Nginx**（VPS 层）
   - SSL 终止
   - `wedding.escapemobius.cc` → `localhost:3080`
   - `/api/` → `localhost:8000`

## 部署步骤

> 详细操作步骤见 [[VPS部署]]

```bash
# 1. 构建镜像
docker-compose build

# 2. 启动服务
docker-compose up -d

# 3. 检查状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f
```

---

## 安全要点

- 数据库端口**严禁暴露**公网，仅通过 Docker 内部网络访问
- 用户面部数据 24 小时自动销毁
- API 接口全部鉴权
- VPS 防火墙仅开放 80/443/22 端口
