## 2. 主指标（已定）

| 层级 | 指标 | 定义 | MVP 目标（首两周后可用数据回调） |
|------|------|------|----------------------------------|
| **工程周会主指标** | 首轮质检达标率 | 单内各张成片在**第一轮** VLM 即达 `quality_acceptable` 的张数 ÷ 该单生成张数 | ≥ **35%** |
| **履约 guardrail** | 订单可交付比 | 单内标记为「可交付」的张数 ÷ **该单 SKU 约定张数** | **P50 ≥ 70%**，**P90 ≥ 50%** |
| **商业 guardrail** | 单均贡献毛利率 | （实收 − 本单可变成本）÷ 实收；可变成本定义见下文 **附录 A** | 不低于 **§4** 表中该档「最低毛利率」 |

**说明**：工程主指标驱动「管线是否白堆」；履约比对接用户感知；毛利率防止补贴换增长。

## 3. MVP 对外承诺（可进用户协议草案）

**锚定套餐**：[[商业模式#基础套餐]] **入门版**（99 元 / 10 张 4K / 2 次重拍 / 30 天）。

| 项 | 决策 |
|----|------|
| **可用定义** | 系统 VLM ≥ `quality_acceptable`，且非 `failed` / `needs_human`（或等价状态）；明显技术废片由系统侧重试与修复吸收，**不扣用户重拍次数**。 |
| **数量** | 每单目标 **10** 张成片位；**首跑至少交付 6 张**「可用」。 |
| **用户重拍** | 套餐内 **2 次**重拍额度；细则（按整单 / 按张）须在支付页与协议中写死，与后端额度一致。 |
| **不足补偿** | 用尽重拍后仍不足约定可用张数的，按 **(实付 ÷ 10) × 短缺张数** 退还差价，**每单退还不超过实付**。 |
| **时效** | 支付成功后 **72 小时内**可下载首版完整结果（提前完成则即时）。 |
| **大面积不可用** | 第三方模型/质检连续不可用导致订单无法完成：不消耗用户重拍额度；订单标记延期；**补偿细则**（券/延期天数）与法务对齐后写入协议——本版占位 **每满 24h 未恢复补偿 5 元券**，设上限。 |

### 3.1 工程与商业「一张皮」义务（强制）

当前实现若仍为「每单 4 张成片」（`PHOTOS_PER_PACKAGE` 等），与上表 **10 张** 冲突。**开启收费前必须二选一**：

1. **改工程**：每付费订单产出与 SKU 一致的张数（如入门版 10 张）；或  
2. **改商业**：对外套餐改为「每单 4 张」并同步改 [[商业模式]]、定价与前端展示。

**未对齐前不得按 10 张套餐收款。**

## 附录 A：单位经济公式（成本口径）

用简单**财务/算术**模型把「套餐价」和「成本」钉在同一页纸上（数字需财务/实测校准）。**避免双算**：同一次质检、修复、重生调用只能计入成本一次。

**两种等价写法，任选其一（勿混用）：**

1. **按调用明细加总（本稿默认，适合做台账）**  
   **每单可变成本** ≈ `Σ（本单内每一笔计费 API 调用 × 单价）`（生成、多轮 VLM、Nano/GPT 修复、失败触发的重跑等均逐笔列清，每笔只进一次）+ 支付通道费 + 存储/带宽（若显著）。

2. **按「单张可交付成片」全链路期望摊销**  
   先估 **`C_photo`**：从该张进入流水线到标记为「可交付」为止，所含全部模型调用的**期望货币成本**。  
   **每单可变成本** ≈ `C_photo × 本单可交付张数` + 通道费 + 存储。**不得**在已用 `C_photo` 的同时再叠加「Σ 质检/修复」。

**每单贡献毛利** ≈ 套餐实收 − 每单可变成本 − 分摊人工/客服（若有）。

与 [[商业模式]] 档位、重拍次数对齐后，可反推：在目标毛利率下，**每张可交付图**的全链路 API 预算上限。
