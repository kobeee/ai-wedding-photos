# PRD分析 - AI高定婚纱摄影 V1.0

## 产品定位

垂直于婚纱摄影领域的专业级 AI 生图 Web 平台，核心卖点是 **"傻瓜式操作 + 影楼级画质"**。

技术核心依赖 [[双模型协作]] 架构：[[Nano-Banana-Pro]] 主攻画质，[[GPT-Image-1.5]] 处理情绪重绘。

## 目标用户

1. **备婚期情侣（主力）** - 95后/00后，预算有限，时间紧迫
2. **社恐/懒人群体** - 排斥镜头前摆拍
3. **极致视觉追求者** - 想要极光、深海、赛博朋克等现实无法完成的场景

> 用户画像直接影响 [[设计系统]] 的风格选择和 [[页面清单]] 的交互设计。

## 核心技术管线（5个模块）

### 模块一：数字人质押（特征提取层）
- 用户上传 5-10 张高清日常照
- 系统提取 Face-ID Embeddings
- "AI试妆间"：生成 3 张测试图供选择，锁定美化权重
- 对应前端页面：上传页 → 试妆页，详见 [[用户旅程]]

### 模块二：中枢调度（摄影导演层）
- 用户选择"视觉套餐"
- LLM 将套餐转化为 Camera Schema（镜头语言 + 光影逻辑）
- 自动编译为底层 Prompt 指令

### 模块三：空间控制与基础生成（物理防撞层）
- ControlNet 骨骼图 + Depth 深度图
- [[Nano-Banana-Pro]] 渲染 4K 底图
- 相关 API 调用方式参见 [[Nano-Banana-Pro#文生图]]

### 模块四：AI视觉反思与自我修复（质检暗房层）⭐核心壁垒
- VLM 找茬：检查手指、穿模、表情
- 物理错误 → [[Nano-Banana-Pro]] 局部重绘
- 情绪错误 → [[GPT-Image-1.5]] I2I 重绘
- 最多循环 3 次直至合格
- 完整流程参见 [[VLM质检管线]]

### 模块五：魔法后期与工业输出（交付层）
- 展示成片 + 魔法笔刷微调
- 8K 超分放大输出

## 用户操作路径（5步）

> 详细交互链路分析见 [[用户旅程]]

1. **建立档案** - 上传男女双方各 5 张日常照
2. **AI试妆** - 选择素颜/精致妆/骨相微调
3. **挑选套餐** - 刷卡片式浏览，无文本输入
4. **沉浸式等待** - 30-60 秒，文案安抚
5. **审片与交付** - 大图呈现 + 下载/换装/重拍

## 商业模式

> 详见 [[商业模式]]

- 免费体验：1 张带水印标清图
- 基础套餐：99/199 元，20 张 4K + 2 张 8K
- SVIP 订阅：599 元/季度，全站解锁 + 无限重绘
- 实体冲印增值：500-1000 元

## MVP 范围（Phase 1）

跑通单模型链路，手工质检，完成 0 号样本打样。

## 对前端设计的启示

> 这些原则贯穿于 [[设计系统]] 和 [[设计规范]] 的每一个决策

- 全流程"去AI化"，隐藏所有技术参数
- 像刷小红书一样的套餐浏览体验
- 沉浸式等待动画很重要（情绪价值）
- 大图震撼呈现是关键转化节点
- 面向年轻用户，设计要高端但不冷淡
