# 2026-03-31

## 今日新增问题确认

这轮没有继续改代码，而是先把当前问题重新定性，避免后续围着错误结论继续修。

### 1. 上传页示意图仍然不合格

- 双人合照示意图看不到完整全身
- 半身示意图看不到脸
- 全身示意图里人物主体甚至不明显
- 这说明当前示意图方案无论素材来源还是裁切策略都不成立

当前结论：

- 问题不只是“不要抽象线稿”
- 也不是简单换成真人图就算完成
- 上传页示意图必须按坑位目标单独产出：
  - `couple_full` 必须明确看到两个人的完整头脚
  - `groom_portrait / bride_portrait` 必须明确看到脸部和半身取景
  - `groom_full / bride_full` 必须明确看到单人完整全身

### 2. 参考图数量是否固定为 5 张，需要重审

- 当前方案是：
  - 必传 4 张
  - 推荐 1 张合照
- 但这是否已经是最终稳定方案，目前不能下结论

需要重新讨论的问题：

- 5 张是否真的足够支撑身份、身形和双人关系
- 是否还需要补充更强的单人近脸图
- 是否要在“最低可用数量”和“最佳效果数量”之间做分层表达
- 前端固定坑位设计，是否已经过早把产品策略写死

### 3. 模特人种要求需要写成明确约束

当前问题不是抽象的“人种偏差”，而是要求写得不够硬：

- 模特参考必须是中国人
- 不接受“东亚脸大概差不多”这种模糊表述
- 当前示意图/样片里明显偏欧洲人，这和产品目标不一致

后续涉及：

- 上传页示意图
- 套餐样片
- 试妆兜底图
- 视觉稿素材

都必须明确检查“中国人”约束，而不是只看“是否像亚洲人”。

### 4. `page.pen` 已明显落后于前端实现

当前实际情况：

- 上传页前端已经不是早期的简单分区上传
- 但 `page.pen` 还没有同步到当前真实结构与文案状态

这会带来两个问题：

- 设计稿无法继续作为前端改动的真源
- 后续任何 UI 调整都容易在“代码现状”和“视觉稿现状”之间反复打架

当前结论：

- 涉及前端 UI 改动时，应优先回到 `page.pen`
- 先把视觉稿修到和当前产品判断一致
- 再让代码跟视觉稿继续收敛

## 当前未闭合事项

- Upload 页示意图方案需要重做，不能沿用当前裁切图
- 参考图数量策略需要重新评估，不应先入为主定死
- 全站人物样片要补“中国人”硬约束
- `page.pen` 需要优先补齐，否则设计链路失真

## 建议的后续顺序

1. 先回改 `page.pen`，把 Upload 页结构、文案、示意图要求重新定稿
2. 明确“最低参考图数量 / 推荐参考图数量”的产品策略
3. 统一梳理所有前端视觉样片，去掉偏欧洲人的素材
4. 再进行下一轮前端 UI 实装

## 2026-03-31 夜间补记：视觉稿已先行定稿

这轮先不碰前端代码，直接把 `page.pen` 回到“设计真源”的位置。

### 本次在视觉稿里已拍板的点

- Upload 页不再沿用旧的“双栏海投上传”结构，改为：
  - 1 个 `双人全身合照` 推荐坑位
  - 4 个必传坑位：`新郎正脸半身 / 新郎全身 / 新娘正脸半身 / 新娘全身`
- 数量口径改成：
  - `最低可用：4 张必传`
  - `推荐增强：补 1 张双人全身合照`
  - 不再把“固定 5 张”写成不可动摇的最终策略
- Upload 页文案里直接写明：
  - 示意图只看取景，不看穿搭
  - 示例人物统一为中国模特
  - 原图直传，服务端保留原始文件

### 本次已补进 `page.pen` 的视觉素材

- Upload 5 个坑位示意图已改为按坑位目标单独生成：
  - `couple_full` 明确双人全身
  - `groom_portrait / bride_portrait` 明确半身看脸
  - `groom_full / bride_full` 明确头脚完整
- Landing Hero、首页推荐样片、套餐展示样片已替换为中国人素材
- Makeup 试妆卡、Package 选择卡、Review 审片大图也已补成中国人素材

### 当前结论

- `page.pen` 现在已经重新领先于当前前端 Upload 结构，可再次作为 UI 改动真源
- 后续前端若继续收敛，应以这版视觉稿为准，而不是继续在旧 Upload 代码上局部打补丁

## 2026-03-31 深夜补记：基于比例锚定再次修正 Upload / Package

在完成第一轮视觉回稿后，又针对“合照是否非必要”“示意图是否看得到完整比例”“场景选择是否层级过浅”做了第二次修正。

### 1. Upload 页口径从 `4 + 1 推荐` 改为 `5 张标准采集`

这次修正不是视觉偏好，而是为了和 [[身材比例锚定方案]] 保持一致：

- 双人全身合照不再写成“有的话尽量传”
- 视觉稿明确改为 `5 张标准采集`
- `双人全身合照` 作为双人比例锚点，用来稳定：
  - 身高差
  - 站姿关系
  - 同框比例

后续前端与后端都应对齐同一逻辑，不应再出现设计写必传、实现写可选的分裂。

### 2. Upload 页补进“原图查看 / 原图预览”的实现指引

为避免固定窗口裁切把脸或全身截没，这轮视觉稿新增了明确提示：

- 示例图支持“点开看原图”的交互语义
- 用户上传后应展示“原图预览”
- 预览表达要优先保留完整比例，而不是继续用默认裁切

这部分虽然还只是视觉稿，但已经是在给后续研发定锚：

- 前端需要提供查看原图或等价交互
- 后端/接口返回也要支持前端展示用户原图

### 3. Package Select 从单层风格卡片升级为三级结构

原先“只看卡片挑风格”的层次太浅，难以承载中式/西式之下的真实细分需求。

这轮改为：

- 第一层：`体系`
  - 中式 / 西式 / 旅拍 / 夜景 / 影棚 / 幻境
- 第二层：`服饰`
  - 旗袍 / 中山装 / 秀禾龙凤褂 / 古朝代 / 主纱 / 缎面礼服
- 第三层：`场景`
  - 海边 / 草原 / 树林 / 古镇 / 城市夜景 / 雪境

样片卡片继续保留，但角色从“唯一选择入口”变为“组合结果预览”。

### 当前定论

- `page.pen` 的 Upload 逻辑现已改成 `5 张标准采集`
- Upload 页已经在视觉层面写明“原图查看 / 原图预览 / 完整比例展示”的研发方向
- Package 页已经从“看图选风格”升级为“体系 → 服饰 → 场景”的产品结构
- 后续代码实现应以这轮视觉稿为准继续收敛
