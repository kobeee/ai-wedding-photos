# 2026-03-28

## 今日结论

- 套餐结构先按试跑版本收敛：
  - `399 = 3 景 / 24 张`
  - `699 = 5 景 / 40 张`
  - `999 = 7 景 / 56 张`
- `699` 作为唯一主推款
- 免费体验保留，先出 `3 张`样片
- 免费体验不把“水印”作为核心策略
- 当前对外交付口径收敛为 `4K`
- 支付第一期优先支付宝，目标是尽快形成真实收费闭环

## 今日统一判断

这轮最重要的不是继续抽象讨论“套餐文案怎么卖”，而是先把系统主线改正确：

`匿名身份 -> 订单 -> 支付 -> 权益生效 -> 生成批次 -> 交付页 -> 售后`

原因：

- 当前系统还停留在 `user_id + session_token + task` 的试玩结构
- 还没有订单、支付、交付、售后四层对象
- 当前 Waiting 页仍会直接调用 `/api/generate`
- 当前工程真实默认产出张数也与未来套餐承诺不一致

在这种状态下，套餐再怎么设计，收费闭环都不稳。

## 今日沉淀

- 新增 [[收费闭环系统改造实施方案]]
- 把账户、订单、支付、履约、交付统一到同一份实施文档中
- 把本轮已拍板决策写入知识库，避免后续继续反复讨论同一层问题

## 接下来执行重点

1. 先做身份与 session 改造
2. 先补订单与 SKU 层
3. 先接支付宝闭环
4. 再把生成入口切到订单驱动
5. 最后再做大张数履约能力

## 备注

- 产品规则、协议、服务条款、退款细则这组内容已确认“必须补”，但本轮先不作为系统改造阻塞项。
- 后续应单独整理为：
  - 用户协议
  - 隐私政策
  - 敏感个人信息授权说明
  - 支付与退款规则
  - 履约与补偿规则

## 晚间实施落地

### 已完成的系统改造

- 后端新增收费闭环底座：
  - `identities`
  - `sessions`
  - `skus`
  - `orders`
  - `payment_transactions`
  - `generation_batches`
  - `deliverables`
- 上传链路不再接受前端回传 `user_id` 复用账户：
  - 优先复用当前 cookie 会话
  - 没有会话时才创建匿名身份
- 鉴权从“只认 Bearer”升级为：
  - 优先 `HttpOnly` session cookie
  - 保留 Bearer 兼容回退
- 文件访问不再直挂公开静态目录：
  - 新增受保护 `/api/files/{kind}/{user_id}/{filename}`
  - 仅资源所属会话可访问
- 新增订单 / 支付 / 履约接口：
  - `GET /api/skus`
  - `POST /api/orders`
  - `GET /api/orders`
  - `GET /api/orders/{order_id}`
  - `GET /api/orders/{order_id}/batches`
  - `GET /api/orders/{order_id}/deliverables`
  - `POST /api/orders/{order_id}/start`
  - `POST /api/orders/{order_id}/reruns`
  - `POST /api/pay/mock/create`
  - `POST /api/pay/mock/confirm`
- 生成链路已切到订单驱动：
  - 未支付订单禁止启动履约
  - 批次状态、进度、消息写入 `generation_batches`
  - 结果图写入 `deliverables`
  - 订单根据交付数量更新 `fulfillment_status`

### 前端已完成的流程改造

- Workflow 状态不再保存真正鉴权凭证，只保留流程上下文
- 主路径已从：
  - `上传 -> 试妆 -> 选套餐 -> 直接生成`
- 改为：
  - `上传 -> 试妆 -> 选视觉主题 -> Checkout 选 SKU -> Pay Result -> Waiting 启动订单履约 -> Review 订单交付`
- 新增页面：
  - `/checkout`
  - `/pay/result`
- Waiting 页不再创建裸任务：
  - 改为显式启动订单批次
  - 轮询订单 / 批次 / 交付物
- Review 页已升级为订单交付页：
  - 展示订单号
  - 展示已交付张数 / 承诺张数
  - 展示剩余重拍额度
  - 支持发起 `rerun`
- Landing 页价格与交付文案已同步到：
  - 免费体验 `3 张`
  - 付费 SKU `399 / 699 / 999`
  - 统一 `4K` 对外交付口径
