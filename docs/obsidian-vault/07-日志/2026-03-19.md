- 用户反馈线上 `https://wedding.escapemobius.cc/` 仍然显示旧版页面，且插图资源未出现
- 排查结论不是资源缺失，而是 VPS 上同时残留了两套婚纱站容器：
  - 旧栈：`wedding-photos-frontend-1` / `wedding-photos-backend-1`

- 根因：
  - 上次部署已成功将新代码同步到 `/opt/apps/wedding-photos/`
  - 新前端镜像也已构建成功，且容器内 `/usr/share/nginx/html/images/` 包含全部 `generated-*.png` 插图资源
  - 但公网端口 `3080` 仍被旧容器 `wedding-photos-frontend-1` 占用，导致流量继续命中旧栈
  - 旧栈对应的根目录 `docker-compose.yml` 文件已不在仓库中，因此常规 `docker compose down` 无法自动清理这批残留容器

## 后端补齐 + Agent-Ready 架构（v0.2.0）

### 目标
从"被动 REST API"转型为"主动暴露能力的 Agent"，同时补齐 AI 管线。

### 三条并行流

#### Stream A：AI 管线补齐
- **新建 `services/director.py`** — LLM 摄影导演服务
  - 套餐语义 → CameraSchema（scene/lighting/composition/lens/mood/wardrobe/pose）→ 渲染 Prompt
  - 调用 laozhang.ai 的 `gemini-3.1-pro-preview` 做 prompt 工程与多模态分析
  - 10 个套餐的完整 fallback 方案（API 不可用时降级）
- **重构 `services/vlm_checker.py`** — 质检结果增加错误分类
  - `IssueCategory.physical`（手指、穿模、光影）→ Nano 局部重绘
  - `IssueCategory.emotional`（表情、眼神、氛围）→ GPT I2I 修复
  - 向后兼容纯字符串 issues，自动猜测分类
- **重写 `routers/generate.py`** — 完整生成管线
  - Director 集成替代硬编码 prompt
  - 3 轮双路径修复循环（physical→Nano, emotional→GPT）
  - 质量评分四档决策：≥0.95 直接交付 / ≥0.85 合格 / ≥0.70 修复 / <0.70 重新生成
  - `GenerateRequest` 增加 `makeup_style`/`gender` 字段，打通妆造→生成流程

#### Stream B：ACP Agent 层 + OpenClaw Skill
- **`acp/` 目录** — 基于 acp-sdk 的 Agent 服务
  - `wedding-photographer`：完整拍摄流程（Director→生图→质检→修复）
  - `makeup-artist`：3 种妆造风格预览
  - `quality-inspector`：VLM 质检报告
- **`skills/wedding-photographer/SKILL.md`** — OpenClaw Skill Manifest
- **docker-compose 新增 `acp` 服务**，独立容器端口 8001

#### Stream C：数据层 + 鉴权
- **`models/database.py`** — aiosqlite 持久化
  - `users` 表：user_id, session_token, photos_count
  - `tasks` 表：task_id, user_id, status, quality_score, result_urls
- **`middleware/auth.py`** — Session Token Bearer 中间件
  - 上传时自动创建用户并返回 `session_token`
  - 公开路径白名单：health, upload, packages, 静态文件

### 部署踩坑（7 个问题）

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | `LoopSetupType` 不存在 | uvicorn 0.35+ 改名 | 锁定 `uvicorn==0.34.0` |
| 2 | `/agents` 返回空列表 | `__main__` 双重导入，两个 Server 实例 | 用 `acp/__main__.py` 做入口 |
| 3 | nginx `host not found in upstream` | Docker DNS 未就绪时 nginx 启动 | `resolver 127.0.0.11` + `set` 变量延迟解析 |
| 4 | `port 3080 already allocated` | docker-proxy 残留进程 | `fuser -k 3080/tcp` 或 `systemctl restart docker` |
| 5 | `asyncio.run() cannot be called from running loop` | `server.run()` 内部自带 `asyncio.run()` | 同步调用，不能 `await` |
| 6 | metadata 参数类型错误 | acp-sdk 需要 `Metadata` 对象不是 dict | 用 `Metadata(tags=..., capabilities=...)` |
| 7 | ACP 容器内无法访问 | 默认绑定 `127.0.0.1` | 改为 `host="0.0.0.0"` |

### 文件清单

修改 7 个：schemas.py, config.py, requirements.txt, vlm_checker.py, generate.py, upload.py, main.py, docker-compose.yml, nginx-frontend.conf

新建 10 个：director.py, database.py, middleware/auth.py, acp/__init__.py, acp/__main__.py, acp/server.py, acp/agents/{photographer,makeup,inspector}.py, SKILL.md
