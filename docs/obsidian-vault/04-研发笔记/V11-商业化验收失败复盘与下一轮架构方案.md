# V11-商业化验收失败复盘与下一轮架构方案

## 结论先行

本轮已经完成：

- Harness 改造同步进生产系统，而不是停留在实验脚本
- VPS 实际部署、重建、健康检查
- 真实 Harness 线上生图复测
- 真实订单全链路验证（上传 → 下单 → 支付 → 生成 → 交付）

但最终结论仍然是：

- **不通过商业化验收**
- 继续在 Prompt 上磨细节已经不是主方向
- 下一轮必须转向 **架构约束**，把“可验证性”做成管线的一等公民

## 本轮线上验证结果

### 1. Close Harness 结果

case:

- `iceland_iceland_intimate_couple`

主要演化：

1. 初版：新娘纯侧脸，身份不可验证
2. 第二版：从单侧脸变成双人对视双侧脸，仍不可验证
3. 第三/四版：成功拉到“双脸更朝向镜头、双眼可见、少 profile”

关键结论：

- Prompt 收紧后，**可读性确实提升**
- 但 close 半身图天然不适合作为“像本人”的主验收图
- 即使正脸更多，只要缺少足够面部面积和比例信息，仍无法给出商用级“像本人”背书

### 2. 真实订单全链路结果

真实线上订单：

- `order_id = ord_5a01e03893c64fb0`
- 套餐：`french`
- entitlement 临时压到 `1 photo`
- 最终状态：`delivered`
- batch 综合分：`0.90`

真实交付图特点：

- wide / full-body，身高比例终于进入可验证区
- 叙事和氛围优于 close harness
- 但人物脸偏小、逆光偏强、五官细节不足

关键结论：

- **wide 图解决了“比例不可验证”**
- 但 **face area 不够大时，身份仍不可验证**
- 这说明只做单轨成图仍然不够，必须拆成“验证图”和“主图”

## 隔离审片结论

本轮严格使用隔离 agent 审片，避免主上下文污染。

### Close harness 审片结论

- 仍然 `不通过`
- 原因：
  - 新娘/新郎身份一致性仍不足以被可靠证明
  - 半身构图无法验证身高和整体体态
  - 更适合氛围图，不适合作为定制主交付

### 真实 wide 订单审片结论

- 仍然 `不通过`
- 原因：
  - 身高/比例可信度达到 `A`
  - 眼神关系与叙事达到 `A`
  - 但人脸太小、逆光太重，像本人只能给 `C`
  - 结果可以作为氛围图，但不适合作为“对标参考人像”的主交付

## 为什么继续磨 Prompt 已经不对

当前问题不是“Prompt 不够严”，而是任务本身互相冲突：

1. 同一张图同时承担：
   - 场景美感
   - 双人关系
   - 身高比例
   - 双脸相似度
   - 妆造完成度

2. close 图天然适合情绪，不适合验身份与比例

3. wide 图天然适合验比例，不适合验五官

4. 因此“要求同一张图同时拿满所有分”本身就是结构不合理

结论：

- 需要把“生成图”和“验收图”解耦
- 需要把“结构验证”和“人脸锁定”解耦
- 需要把“主图交付”和“隐藏质检图”解耦

## 下一轮正确方向：架构升级，不是提示词微调

### 方案总纲：双轨生成

把每张商业成片拆成两条轨道：

1. **Validation Track**
   - 强制生成一张同框全身或至少 3/4 身图
   - 要求：
     - 两人都清晰可见
     - 身高差、头身比、肩线关系可验证
     - 双脸至少达到可核验清晰度
   - 这张图不一定交付给用户，但必须通过内部质检

2. **Hero Track**
   - 生成真正的氛围主图
   - 可以 close / medium
   - 但必须继承 Validation Track 已通过的身份与比例约束

### 方案总纲：分人锁脸

在当前四轮之后，新增“按人分开的局部身份锁定”：

1. face crop 新娘
2. face crop 新郎
3. 分别对两张脸做高权重 identity repair / local redraw
4. 再回写到主图

目的：

- 不再让模型一次同时处理双脸竞争
- 提高单人五官保真
- 减少“其中一人像、另一人漂”的不稳定性

### 方案总纲：验收门槛前移

把下面这些改成真正的硬门槛：

- 若主脸面积太小 → 直接不允许作为主交付
- 若双人高度关系不可验证 → 直接不允许作为主交付
- 若一方脸只剩弱 3/4 甚至侧脸 → 直接不允许作为“身份主图”
- 若 hero 图人脸不足以证明相似度，则只能降级为氛围补图，不算核心交付 quota

## 下一轮具体实施任务

### Task 1. 定义“可验证图”标准

新增内部标准字段：

- `is_identity_verifiable`
- `is_proportion_verifiable`
- `face_area_ratio_bride`
- `face_area_ratio_groom`
- `body_visibility_score`

其中任一不达标：

- hero 图不能直接交付

### Task 2. 新增 Validation Track

在 `editorial_pipeline` 之外再包一层 orchestration：

1. 先跑 validation render
2. VLM 检查：
   - 身高顺序
   - 头身比例
   - 双脸可验证性
3. 验证通过后，再生成 hero render

### Task 3. 新增 Face Lock Pass

新加一轮或两轮：

- `R5 bride_face_lock`
- `R6 groom_face_lock`

实现方向：

- 允许对主图做人脸局部重绘/局部修复
- 两人分开处理，避免注意力竞争

### Task 4. 交付分层

把交付图分成：

- `validation-safe`：比例/身份可验证图
- `hero-atmosphere`：氛围主图

商业交付要求：

- 每一单至少要有 1 张 `validation-safe`
- 没有这张图，则整单不能算通过

### Task 5. Harness 升级

新 Harness 不能只看单图美感，而要看：

- close hero
- medium hero
- wide validation
- face-lock 后成片

并且审片维度必须拆成：

- `identity_verifiability`
- `proportion_verifiability`
- `hero_quality`
- `commercial_pass`
