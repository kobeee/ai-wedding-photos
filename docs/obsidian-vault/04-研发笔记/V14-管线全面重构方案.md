# V14 管线全面重构方案

> 日期：2026-04-06
> 状态：方案确定，待实施
> Codex 评审：B+（二轮 battle 后采纳 3 项建议）

## 背景

V12-V13 管线经过多轮 harness 验证，暴露出根本性的架构问题：
- **四轮法**（R1场景→R2细修→R3身份融合→R4妆造）= 4 次 Nano 调用/轮
- **双轨制**（validation + hero 各跑一遍）= 8 次起步
- 单张成本 ~$1.56，典型场景 20~40 次 API 调用
- Hero track 成功率仅 67%
- **核心病因**：R1 不带参考图 → R3 hack 合脸 = "先画再改"路径

## 核心改动

### 1. 四轮法 → 两轮法

| 轮次 | 做什么 | 关键改变 |
|------|--------|---------|
| R1 新 | 带 5 张参考图 + 场景 prompt，`multi_reference_generate` 一次性生成 | 模型生成阶段就有身份信息，不是后期合脸 |
| R2 新 | 合并细修+妆造，`repair_with_references` | 一轮搞定，不再分两步 |

### 2. 双轨制 → 单轨制

- 删除 validation/hero 分叉
- medium/medium-wide 构图兼顾身份和氛围
- face_lock 作为后置修复兜底

### 3. VLM 质检 + face crop（Codex 建议）

- 质检时裁切人脸区域放大做身份比对
- 不额外调 API，零成本

### 4. 场景重设计

淘汰 6 个（iceland/cyberpunk/starcamp/artistic-fantasy/travel-destination/onsen）
保留 4 个（french/minimal/chinese-classic/western-romantic）
新增 6 个（garden-editorial/blue-hour/film-romantic/new-chinese/seaside-golden/greenhouse）

### 5. 其他

- 阈值先不动，跑数据再调（Codex 建议）
- 旧管线 fallback 开关：`config.pipeline_version`（Codex 建议）
- 重试初始设 2，按数据调整
- 参考图传入时 prompt 明确标注男女顺序

## 预期效果

| 指标 | V13 | V14 |
|------|-----|-----|
| Nano 调用/张 | 8~40 | 2~6 |
| 成本/张 | ~$1.56 | ~$0.30~0.50 |
| 生成耗时 | 237~795s | 20~60s |

- 认为重试 5→2 过激

### 反驳后二审：B+

- 接受一步到位 + harness 验证

1. face crop 质检（裁切人脸放大做身份比对）
2. 阈值先不动（跑数据再调）
3. fallback 开关（config 层切换，秒回滚）

### 拒绝的建议
- 三轮法过渡（❌ 旧思路修补）
- 分步实施（❌ 一步到位更快）
- mask 传参（❌ Nano API 技术上不支持）
