## 二、新用户旅程

```
Landing → /plan（选套餐+邮箱） → /scenes（选景） → /upload（上传5张）
→ /makeup（妆造） → /waiting（自动建单+生成） → /delivery/:orderId（交付）
                                                 ↗ /orders（邮箱找回）
```

### 与 V8 对比

| 维度 | V8 | V9 |
|------|----|----|
| 入口 | 直接上传 | 先选套餐+留邮箱 |
| 选景 | 选套餐时内嵌 | 独立页面，按类目分组 |
| 计费 | Checkout 页手动选 SKU | PlanSelect 页统一选 |
| 订单创建 | Checkout 页创建 | Waiting 页自动创建 |
| 交付 | /review（无参数） | /delivery/:orderId |
| 找回 | 无 | /orders 邮箱查询 |
| 体验码 | 无 | 免费套餐需体验码 |

## 三、后端变更

### 新增数据表

1. **scene_catalog** — 10 个场景，6 个类目（western/chinese/travel/studio/night/fantasy）
2. **order_scene_selections** — 订单-场景多对多关联
3. **experience_codes** — 体验码（code + channel + status + 绑定信息）

### 新增 API

| 端点 | 方法 | 鉴权 | 用途 |
|------|------|------|------|
| `/api/plans` | GET | 公开 | SKU 列表（/api/skus 别名） |
| `/api/scenes` | GET | 公开 | 场景目录 |
| `/api/experience-codes/verify` | POST | 公开 | 验证体验码 |
| `/api/orders/lookup?email=` | GET | 公开 | 邮箱找回订单 |

### orders 表迁移

- 新增字段：`email TEXT DEFAULT ''`、`experience_code_id TEXT`、`delivery_status TEXT DEFAULT 'pending'`
- 新增索引：`idx_orders_email`
- `create_order` 签名扩展：接受 `email`、`scene_ids`、`experience_code_id`
- 自动推导：第一个 scene 的 `brief_ref` → `package_id`

## 四、前端变更

### 新增页面（4 个）

| 页面 | 路由 | StepHeader | 核心功能 |
|------|------|-----------|---------|
| PlanSelect | /plan | current=1 | SKU 卡片选择 + 邮箱 + 体验码 |
| SceneSelect | /scenes | current=2 | 场景网格，按类目分组，配额控制 |
| Delivery | /delivery/:orderId | — | 交付预览 + 下载 + 重拍（替代 Review） |
| Orders | /orders | — | 邮箱查询订单列表 |

### 改造文件（6 个）

| 文件 | 变更 |
|------|------|
| `App.tsx` | 路由重写：/plan, /scenes, /delivery/:orderId, /orders |
| `StepHeader.tsx` | 6 步：选套餐→选景→上传→妆造→等待→交付 |
| `workflow.ts` | 新增 email, selectedSceneIds, experienceCode 字段 |
| `Landing.tsx` | CTA 指向 /plan，footer 新增订单查询 |
| `Upload.tsx` | StepHeader current=3 |
| `Makeup.tsx` | StepHeader current=4，下一步指向 /waiting |
| `Waiting.tsx` | 自动建单逻辑 + 导航到 /delivery/:orderId |

### Waiting 页自动建单流程

```
1. fetchLatestOrder() → 有订单？直接进入轮询
2. 无订单 → 检查 workflow 状态（selectedSku + selectedSceneIds）
3. 状态齐全 → createOrder() → 免费单直通生成，付费单跳转支付
4. 状态不全 → 重定向到 /plan
```

### 废弃页面

- `PackageSelect.tsx` → 被 PlanSelect + SceneSelect 取代
- `Checkout.tsx` → 建单逻辑合并到 Waiting
- `Review.tsx` → 被 Delivery.tsx 取代（加了 orderId URL 参数）

## 六、遗留与后续

### 已知限制

1. **多场景生成**：当前只用第一个 scene 的 brief_ref 做 package_id，多场景订单只会按第一个场景生成。需要迭代 `generate_order_batch` 实现多场景轮流生成。
2. **场景预览图**：scene_catalog 的 `preview_url` 全为空，前端用渐变色 fallback。需要为每个场景准备高质量预览图。
3. **废弃页面清理**：PackageSelect.tsx、Checkout.tsx、Review.tsx 仍在代码库中但不再被路由引用，可择机删除。
4. **付费流程**：PlanSelect → 付费 SKU → Waiting 自动建单 → 跳转支付 → PayResult → 回到 Waiting。链路代码已写但未实际走通付费（需要支付提供商对接）。
