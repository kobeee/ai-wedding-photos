# V10-四轮统一管线与妆造终修落地

## 背景

线上生产链路长期停留在“一步多参考直出 + VLM 返修”，而 Harness 已经验证过更优的多轮生成路径。两者脱节导致：

- Harness 高分不代表线上真人观感高分
- 身高比例与双人站位没有被结构锚单独控制
- 最终成片没有继承用户在试妆阶段选中的具体妆造图
- VLM 的 `identity_match` 曾经在没有参考脸输入的前提下盲判

## 本轮改动

### 1. 四轮统一

新管线统一为：

1. `R1` 场景/结构锚定
   - 若有 `couple_full`，优先只拿它做双人比例与站位锚
   - 不再让模型一上来同时扛场景 + 双脸 + 妆造
2. `R2` 细节精修
   - 只修服饰、手部、皮肤与眼神可读性
   - 构图锁死
3. `R3` 身份融合
   - `R2` 成片作为构图锚
   - 新郎/新娘身份参考单独输入，尽量排除 couple anchor 干扰
4. `R4` 妆造终修
   - 把用户在试妆页选中的预览图重新引回最终成片
   - 只做妆造/肌理/眼神收口，不再改比例和构图

### 2. 参考图语义拆分

- `structure_refs`：专供 R1 锁定双人相对比例
- `identity_refs`：专供 R3/R4 锁定真人脸

避免一张 couple 图同时承担“比例��� + 身份锚”而造成相互干扰。

### 3. 视角与眼神控制

Prompt 层新增：

- 少侧脸：优先正面或轻微 3/4 角，避免 hard profile
- 眼神关系显式化：
  - close / medium 双人镜头必须有可读 eye-line
  - 支持“互相对视”或“一人看伴侣、一人看镜头”的明确关系
- 避免：
  - `hard side profile`
  - `extreme profile view`
  - `obscured eyes`
  - `both faces turned away from camera`

### 4. VLM 质检纠偏

`identity_match` 现在会真正看到参考脸：

- 第 1 张图：生成结果
- 后续图：身份参考图

从“盲判像不像”修正为“带参考脸的身份质检”。

### 5. 前端到后端的数据闭环

前端现在除了保存 `selectedMakeup` 风格，还会保存：

- `selectedMakeupReferences.groom`
- `selectedMakeupReferences.bride`

订单启动 / 重拍时把选中的试妆图 URL 一并发给后端，R4 终修会优先使用用户实际选中的预览图。
