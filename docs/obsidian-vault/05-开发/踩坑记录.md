## 部署相关

### 前端配图未显示 — Docker 构建缓存 + Compose 项目名冲突
- **日期**: 2026-03-17
- **分类**: 部署
- **现象**: 设计稿中的 5 张配图（Hero 背景 + 4 张视觉套餐卡片）在线上页面不显示
- **原因（多层）**:
  1. 前端代码根本没引用图片 — `gallery__img` 是空 div，Hero 背景是纯色 `#1A1A1A`
  2. 图片文件在项目根目录 `images/` 下，未复制到 `src/frontend/public/images/`
  3. 修复代码后部署，`docker compose build` 使用了构建缓存，新加的 `public/images/` 没进容器
  4. 从不同目录执行 `docker compose` 导致项目名不一致（`wedding-photos-` vs `docker-`），端口冲突
- **解决**:
  1. 将 5 张图片复制到 `src/frontend/public/images/` 并重命名为语义化文件名
  2. Landing.tsx: Hero 加 CSS 背景图，Gallery 卡片加 `backgroundImage` 内联样式
  3. 部署时使用 `docker compose build --no-cache` 强制重建
  4. 统一用 `-p wedding-photos` 固定 Compose 项目名
- **教训**:
  - 设计稿有图片时，前端实现必须同步引用，不能留空占位
  - Docker 部署涉及静态资源变更时，务必 `--no-cache` 重建
  - `docker compose` 始终指定 `-p` 项目名，避免目录名导致的命名混乱
