## 路由配置

在 `App.tsx` 中通过 React Router DOM v7 配置：

| 路由 | 组件 | 说明 |
|------|------|------|
| `/` | `Landing` | 营销落地页 |
| `/upload` | `Upload` | 照片上传 |
| `/makeup` | `Makeup` | AI 试妆间 |
| `/package` | `PackageSelect` | 套餐选择 |
| `/waiting` | `Waiting` | 沉浸式等待 |
| `/review` | `Review` | 审片与交付 |

SPA 路由需要 Nginx fallback 到 `index.html`，参见 [[Nginx配置]]。

## 组件开发规范

### 文件组织
- 每个页面：`PageName.tsx` + `PageName.css`
- 公共组件：`components/` 目录
- 一个组件一个文件，不做过度拆分
- 静态视觉资源统一放 `public/images/`

### 状态管理
- Phase 1 使用 React 内置 `useState` / `useContext`
- 不引入 Redux/Zustand（复杂度不够高）
- 页面间数据传递通过 URL params + Context

### 与后端对接
- 通过 `fetch` 调用 [[API设计]] 定义的接口
- 请求路径统一使用 `/api/` 前缀
- Nginx 反代到后端，参见 [[Nginx配置]]

## 2026-03-19 移动端适配修复

### 结论

- Phase 1 的 6 个页面已完成基础移动端适配，可在 `390px` 常见手机宽度下稳定阅读和操作
- 本轮修复的重点不是继续缩小字号，而是把错误选择器、头部拥挤和卡片节奏一起纠正

### 关键改动

- 首页 `Landing.css` 修复了一批未命中 DOM 的响应式类名
- `Header` 与 `StepHeader` 增加手机端布局策略，降低顶部信息密度
- `Upload / Makeup / PackageSelect / Waiting / Review` 统一调整：
  - 标题行高
  - 卡片堆叠方式
  - 按钮宽度
  - 触控尺寸
  - 审片页预览区比例

### 本轮涉及文件

- `src/frontend/src/index.css`
- `src/frontend/src/components/Header.css`
- `src/frontend/src/components/StepHeader.css`
- `src/frontend/src/pages/Landing.css`
- `src/frontend/src/pages/Upload.css`
- `src/frontend/src/pages/Makeup.css`
- `src/frontend/src/pages/PackageSelect.css`
- `src/frontend/src/pages/Review.css`
- `src/frontend/src/pages/Waiting.css`

### 经验

- 响应式问题优先排查“类名是否命中真实结构”，其次才是间距与字号
- 手机上套餐页双列很容易变成信息墙，本项目更适合单列沉浸式浏览
- 步骤型流程页不要在手机上硬塞桌面版头部结构
