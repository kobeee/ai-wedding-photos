## AI 管线架构

```
用户上传照片
    ↓
Director（LLM gpt-4o）
    套餐 + 妆造 + 性别 → CameraSchema → 渲染 Prompt
    ↓
Nano Banana Pro（Gemini）
    参考图 + Prompt → 4K 底图
    ↓
VLM Checker（gpt-4o vision）
    质检 → QualityReport（score + issues with category）
    ↓
┌─ score ≥ 0.95 → 直接交付
├─ score ≥ 0.85 → 合格交付
├─ score ≥ 0.70 → 双路径修复 ↓
│   ├─ physical issues → Nano 局部重绘
│   └─ emotional issues → GPT I2I 修复
│   → 回到 VLM Checker（最多 3 轮）
└─ score < 0.70 → 重新生成（回到 Nano）
```

### Director 服务

`services/director.py` — LLM 摄影导演

- 输入：package_id + makeup_style + gender + preferences
- 输出：CameraSchema + 渲染 Prompt
- 10 个套餐的完整 fallback（API 不可用时降级为硬编码方案）
- 套餐列表：iceland, french, cyberpunk, minimal, onsen, starcamp + 4 个旧 ID 兼容

### VLM 质检分类

`IssueCategory` 枚举决定修复路径：

| 分类 | 典型问题 | 修复模型 |
|------|----------|----------|
| `physical` | 手指畸形、穿模、光影错误、边缘瑕疵 | Nano Banana（局部重绘） |
| `emotional` | 表情僵硬、眼神空洞、氛围不匹配 | GPT Image（I2I 编辑） |

## 数据层

### SQLite 表结构

```sql
-- 用户表
CREATE TABLE users (
    user_id TEXT PRIMARY KEY,
    session_token TEXT UNIQUE NOT NULL,
    photos_count INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);

-- 任务表
CREATE TABLE tasks (
    task_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    package_id TEXT DEFAULT '',
    status TEXT DEFAULT 'pending',
    progress INTEGER DEFAULT 0,
    quality_score REAL DEFAULT 0.0,
    message TEXT DEFAULT '',
    result_urls TEXT DEFAULT '[]',  -- JSON array
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);
```

数据库文件：`./data/lumiere.db`，Docker volume `backend-data` 持久化。

## 鉴权机制

Session Token Bearer 中间件（`middleware/auth.py`）：

1. 用户上传照片时自动创建 user + 生成 `lst_` 前缀的 token
2. 后续请求通过 `Authorization: Bearer lst_xxx` 鉴权
3. 公开路径白名单：`/api/health`, `/api/upload`, `/api/packages`, `/docs`, 静态文件

Phase 2 升级 JWT。

## ACP Agent 层

三个 Agent 通过 acp-sdk 注册，独立容器端口 8001：

| Agent | 能力 | 用途 |
|-------|------|------|
| `wedding-photographer` | text-to-image, image-to-image, quality-check | 完整拍摄流程 |
| `makeup-artist` | - | 3 种妆造风格预览 |
| `quality-inspector` | - | VLM 质检报告 |

ACP 端点：`GET /agents`（发现）、`POST /runs`（执行）、`GET /runs/{id}`（状态）

> 关键：入口必须是 `acp/__main__.py`，不能直接跑 `acp/server.py`，否则会触发 Python `__main__` 双重导入问题。详见 [[踩坑记录#Python __main__ 双重导入]]。
