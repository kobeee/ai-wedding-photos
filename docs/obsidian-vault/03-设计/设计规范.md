# 设计规范

> 设计原则与交互规范，指导所有页面的设计实现。

---

## 设计原则

### 1. 去 AI 化

来源：[[PRD分析#对前端设计的启示]]

- 全流程隐藏技术参数（Prompt、模型名、参数面板）
- 用"摄影"语言替代"AI"语言
  - ❌ "AI 正在生成图片"
  - ✅ "摄影师正在为您精心布光"
- 进度展示用情感文案，不用技术日志

### 2. 沉浸式体验

- 深色背景营造影院感
- 大面积留白，呼吸感
- 过渡动画柔和，不突兀
- 等待页面是"体验"而非"忍耐"

### 3. 高端但有温度

- 金色点缀传递奢华感，但不过度
- 文案温暖、贴心，有"人味"
- 圆角柔和（参见 [[设计系统#圆角体系]]），避免尖锐感

### 4. 降低决策压力

- 套餐选择用视觉而非文字
- "刷小红书"式的浏览体验
- 每个步骤只做一件事

## 交互规范

### 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | 金色背景 + 深色文字，胶囊形 | CTA、关键操作 |
| Secondary | 透明背景 + 金色边框 | 次要操作 |
| Ghost | 无边框 + 文字链接 | 辅助操作 |
| Disabled | 灰色，降低透明度 | 不可操作状态 |

```css
/* Primary 按钮 */
background: var(--accent-gold);
color: var(--bg-primary);
border-radius: 100px;
padding: 12px 32px;
font-weight: 500;
letter-spacing: 0.15em;
```

### Hover 效果

- 按钮：亮度提升 + 微弱阴影
- 卡片：上浮 2-4px + 金色描边
- 链接：颜色从 `--text-secondary` → `--accent-gold`
- 所有 hover 过渡 0.3s ease

### 加载状态

- 骨架屏（Skeleton）：深灰色脉冲动画
- 按钮加载：文字替换为旋转图标
- 页面加载：全屏中心 Logo 脉冲

### 表单输入

- 背景色：`--bg-surface`
- 边框：`1px solid var(--border-subtle)`
- 焦点态：金色边框
- 圆角：12px
- 内边距：12px 16px

## 布局规范

### 页面容器

```css
max-width: 1200px;
margin: 0 auto;
padding: 0 24px;
```

### 网格系统

- 12 列网格（桌面端）
- 间距 24px
- 断点：
  - Desktop: > 1024px
  - Tablet: 768px - 1024px
  - Mobile: < 768px

### 响应式策略

Phase 1 以桌面端为主，但预留响应式断点。

- 卡片网格：Desktop 3列 → Tablet 2列 → Mobile 1列
- 侧边栏：Desktop 左右分栏 → Mobile 上下堆叠
- 导航：Desktop 横向 → Mobile 汉堡菜单

## 图标规范

使用 Lucide React 图标库（参见 [[系统架构#前端]]）。

| 图标 | 用途 |
|------|------|
| Upload | 上传区域 |
| Camera | 拍照/生成 |
| Download | 下载 |
| ChevronRight | 下一步 |
| Check | 完成/选中 |
| X | 关闭/删除 |
| Sparkles | AI 魔法相关 |

图标大小：16px（小）/ 20px（中）/ 24px（大）
图标颜色：跟随文字颜色或使用 `--accent-gold`

## 文案规范

### 语气

- 温暖、贴心，像一个体贴的婚礼策划师
- 避免技术术语
- 适度使用 emoji（但不过度）

### 示例

| 场景 | 文案 |
|------|------|
| 上传引导 | "上传你们的日常照片，让 AI 认识你们" |
| 试妆引导 | "选一个你喜欢的妆造风格" |
| 等待安抚 | "每一帧都值得精心雕琢，请稍候..." |
| 生成完成 | "您的专属婚纱照已就绪" |
| 付费引导 | "解锁高清无水印大图" |
