## Landing Page（营销落地页）

**路由**: `/`
**文件**: `pages/Landing.tsx` + `Landing.css`

### 区块结构
1. **Header** - Logo + 导航 + CTA 按钮
2. **Hero** - 800px 高，全幅沉浸背景，主标语 + 副标语 + CTA
3. **五步流程** - 金色编号圆形，横向排列
4. **套餐展示** - 样片网格
5. **定价** - 三栏对比，中间突出（参见 [[商业模式]]）
6. **FAQ** - 手风琴展开/收起交互
7. **底部 CTA** - 再次引导
8. **Footer** - 版权信息

### 设计要点
- Hero 区域 800px 高，打造沉浸感
- 五步流程编号使用 [[设计系统]] 的金色圆形
- 定价栏中间档位视觉突出
- FAQ 手风琴交互

## Upload Page（建立档案）

**路由**: `/upload`
**文件**: `pages/Upload.tsx` + `Upload.css`

> [!success]
> 该页已于 2026-03-31 夜间完成回稿，`page.pen` 已重新成为 Upload 页的设计真源。
> 本轮定稿重点：
> - 结构改为 `5 张标准采集`
> - `双人全身合照` 纳入必传，用作双人比例锚点
> - 示例人物统一为中国模特
> - 5 个坑位示意图已按真实取景目标单独重做，并增加原图查看/原图预览提示

### 核心交互
- 5 坑位上传：新郎半身 / 新郎全身 / 新娘半身 / 新娘全身 / 双人全身合照
- 拖拽 + 点击上传
- 坑位示意图引导
- 上传后显示用户原图预览，并支持点开看原图
- AI 预检提示
- 完成后进入 `AI试妆`

### 设计要点
- Upload 口径明确为：
  - `5 张标准采集`
  - `双人全身合照` 用于稳定身高差、站姿与同框关系
- 示意图只说明取景，不说明穿搭；示例人物统一为中国模特
- 每个坑位都要表达“支持看原图 / 保留完整比例”的实现方向
- 深色卡片 + 金色边框层级，维持高端沉浸基调
- 步骤头部组件 `StepHeader`（当前步骤高亮）

## Package Select（套餐选择）

**路由**: `/package`
**文件**: `pages/PackageSelect.tsx` + `PackageSelect.css`

### 核心交互
- 先选 `体系`，再选 `服饰`，最后选 `场景`
- 样片卡片作为组合结果预览，不再承担全部筛选职责
- 无文本输入，保持视觉驱动选择
- 点击展开大图预览
- 选中后确认

### 设计要点
- 第一层为婚礼语境：`中式 / 西式 / 旅拍 / 夜景 / 影棚 / 幻境`
- 第二层为服饰细分：如 `旗袍 / 中山装 / 秀禾龙凤褂 / 古朝代 / 主纱 / 缎面礼服`
- 第三层为场景细分：如 `海边 / 草原 / 树林 / 古镇 / 城市夜景 / 雪境`
- 视觉稿中需明确展示“体系 → 服饰 → 场景”的产品结构
- 参照 [[设计规范]] 的卡片交互规范
- 卡片 hover 时上浮 + 金色描边
- "无需思考"的选择体验

## Review Page（审片与交付）

**路由**: `/review`
**文件**: `pages/Review.tsx` + `Review.css`

### 核心交互
- 左侧：大图展示区，支持缩放
- 右侧：操作面板
  - 缩略图列表（切换查看）
  - 下载按钮（标清/4K/8K）
  - 换装按钮
  - 重拍按钮
- 付费墙在下载高清时触发

### 设计要点
- 大图 = 关键转化节点
- 下载 CTA 使用金色按钮
- 对应 API：[[API设计#GET /api/generate/{task_id}/result]]
