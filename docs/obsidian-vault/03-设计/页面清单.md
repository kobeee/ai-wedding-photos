## Landing Page（营销落地页）

**路由**: `/`
**文件**: `pages/Landing.tsx` + `Landing.css`

### 区块结构
1. **Header** - Logo + 导航 + CTA 按钮
2. **Hero** - 800px 高，全幅沉浸背景，主标语 + 副标语 + CTA
3. **五步流程** - 金色编号圆形，横向排列
4. **套餐展示** - 样片网格
5. **定价** - 三栏对比，中间突出（参见 [[商业模式]]）
6. **FAQ** - 手风琴展开/收起交互
7. **底部 CTA** - 再次引导
8. **Footer** - 版权信息

### 设计要点
- Hero 区域 800px 高，打造沉浸感
- 五步流程编号使用 [[设计系统]] 的金色圆形
- 定价栏中间档位视觉突出
- FAQ 手风琴交互

## Upload Page（建立档案）

**路由**: `/upload`
**文件**: `pages/Upload.tsx` + `Upload.css`

### 核心交互
- 分区上传：男方 / 女方
- 拖拽 + 点击上传
- 缩略图预览网格
- 上传进度条（金色）
- 完成后激活"下一步"

### 设计要点
- 上传区域虚线边框 + 深色背景
- 进度条使用 `--accent-gold`
- 步骤头部组件 `StepHeader`（当前步骤高亮）

## Makeup Page（AI试妆间）

**路由**: `/makeup`
**文件**: `pages/Makeup.tsx` + `Makeup.css`

### 核心交互
- 三种妆造风格卡片
- 点击选中，金色边框高亮
- 预览图大图展示
- 确认进入下一步

### 设计要点
- 卡片 16px 圆角
- 选中态：金色边框 + 微弱光晕
- 对应 API：[[API设计#POST /api/makeup/generate]]

## Package Select（套餐选择）

**路由**: `/package`
**文件**: `pages/PackageSelect.tsx` + `PackageSelect.css`

### 核心交互
- 卡片瀑布流，类小红书体验
- 无文本输入，纯视觉选择
- 点击展开大图预览
- 选中后确认

### 设计要点
- 参照 [[设计规范]] 的卡片交互规范
- 卡片 hover 时上浮 + 金色描边
- "无需思考"的选择体验

## Waiting Page（沉浸式等待）

**路由**: `/waiting`
**文件**: `pages/Waiting.tsx` + `Waiting.css`

### 核心交互
- 全屏居中等待界面
- 圆形进度指示 + 百分比
- 当前步骤文字提示（轮播文案）
- 预计剩余时间
- 轮询 [[API设计#GET /api/generate/{task_id}/status]]

### 设计要点
- 全屏居中，背景微动效
- 进度条金色渐变
- 安抚文案轮播：
  - "AI 摄影师正在为您精心布光..."
  - "正在调整最完美的角度..."
  - "您的婚纱照即将呈现..."

## Review Page（审片与交付）

**路由**: `/review`
**文件**: `pages/Review.tsx` + `Review.css`

### 核心交互
- 左侧：大图展示区，支持缩放
- 右侧：操作面板
  - 缩略图列表（切换查看）
  - 下载按钮（标清/4K/8K）
  - 换装按钮
  - 重拍按钮
- 付费墙在下载高清时触发

### 设计要点
- 大图 = 关键转化节点
- 下载 CTA 使用金色按钮
- 对应 API：[[API设计#GET /api/generate/{task_id}/result]]
